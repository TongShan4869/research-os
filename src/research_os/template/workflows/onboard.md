# Research OS Onboarding

Use this workflow when the user runs `/research-os:onboard` or asks to initialize their Research OS workspace.

1. Check `research-os.yaml`.
2. Ask whether the user has an existing Obsidian vault.
3. Ask whether Zotero Desktop should be checked.
4. Ask which project roots should be registered.
5. Offer to create the first project.
6. Run `research-os validate`.
7. Run `research-os build-graph` after registry or note changes.
